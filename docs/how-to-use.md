# How to Use — Health Gen AI Chat

This guide walks you from a fresh checkout to a running chat session where you can ask natural-language questions about your health data and get SQL results and charts back.

## Architecture overview

```
You (browser or terminal)
      │
      ▼
Chat Agent  ─── Claude Opus 4.8 (Anthropic API)
      │
      ├── MCP: mcp_semantic   → dbt artifact reader (metrics, columns, lineage)
      ├── MCP: mcp_exec       → read-only MySQL query runner
      └── MCP: mcp_visualization → Vega-Lite chart generator
```

The three MCP servers run as child processes of the chat agent. You only need to start the agent — it manages the rest.

| Requirement | Version | Notes |
|---|---|---|
| Python | 3.11.x | Exactly 3.11; the project sets `requires-python = ">=3.11, <3.12"` |

## Step 1 — Install dependencies

```bash
cd health-gen-ai-chat

# Create and activate a virtual environment (Python 3.11 only)
python3.11 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# Install all packages
pip install -e .
```

This installs all three MCP server packages plus the chat agent and the Anthropic SDK.

---

## Step 2 — Configure dbt

The dbt project lives in `dbt_health_gen_ai_chat/`. It needs a `profiles.yml` to connect to your MySQL instance.

Create `~/.dbt/profiles.yml` (or `dbt_health_gen_ai_chat/profiles.yml`):

```yaml
health_gen_ai_chat:
  target: dev
  outputs:
    dev:
      type: mysql
      server: localhost       # your MySQL host
      port: 3306
      schema: dbt_dev         # schema where dbt creates its models
      username: your_user
      password: your_password
      ssl_disabled: true      # set false if using SSL
```

Test the connection:

```bash
cd dbt_health_gen_ai_chat
dbt debug
```

---

## Step 3 — Compile dbt

The semantic MCP server reads `target/manifest.json` and `target/semantic_manifest.json`. These files are generated by `dbt compile`:

```bash
cd dbt_health_gen_ai_chat
dbt compile
cd ..
```

You should see two files created:

```
dbt_health_gen_ai_chat/target/manifest.json
dbt_health_gen_ai_chat/target/semantic_manifest.json
```

> Re-run `dbt compile` any time you change your dbt models or schema definitions.

---

## Step 4 — Set environment variables

```bash
export ANTHROPIC_API_KEY="sk-ant-..."
export MYSQL_ALCHEMY_URI="mysql+pymysql://your_user:your_password@localhost:3306/health_db"
```

Optional overrides for dbt artifact paths (only needed if your paths differ from the defaults):

```bash
export DBT_MANIFEST_PATH="/absolute/path/to/manifest.json"
export DBT_SEMANTIC_MANIFEST_PATH="/absolute/path/to/semantic_manifest.json"
```

> **Tip:** Put these in a `.env` file and load them with `source .env` or use `python-dotenv`.

---

## Step 5 — Run the chat agent (terminal mode)

```bash
python -m src.chat_agent.main
```

You will see:

```
Health Gen AI Chat — type 'quit' to exit.

You: 
```

Type any question about your health data. The agent will:
1. Call the semantic tools to discover available models and columns.
2. Write and execute a SQL query.
3. If appropriate, generate a chart and print the Vega-Lite spec.

Type `quit`, `exit`, or `q` to stop. Press `Ctrl+C` to exit immediately.

## Step 6 — Use the web UI

The web UI (`src/chat_agent/index.html`) is a standalone HTML file that talks to a backend at `POST /api/chat`. You need a thin HTTP wrapper around the agent to use it.

### Quick backend with FastAPI

Install FastAPI and Uvicorn:

```bash
pip install fastapi uvicorn
```

Create `src/chat_agent/server.py`:

```python
from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles
from fastapi.responses import JSONResponse
from pydantic import BaseModel
import asyncio, os, anthropic
from pathlib import Path

app = FastAPI()

SRC = Path(__file__).parent.parent
SYSTEM_PROMPT = open(Path(__file__).parent / "main.py").read().split('"""')[3]  # reuse from main.py

class ChatRequest(BaseModel):
    messages: list[dict]

@app.post("/api/chat")
async def chat(req: ChatRequest):
    client = anthropic.AsyncAnthropic(api_key=os.environ["ANTHROPIC_API_KEY"])
    mcp_servers = [
        anthropic.types.beta.BetaMCPServerStdioParams(
            name="mcp_semantic",
            command="python", args=["-m", "mcp_semantic_healh_gen_ai_chat.main"], env=dict(os.environ)
        ),
        anthropic.types.beta.BetaMCPServerStdioParams(
            name="mcp_exec",
            command="python", args=["-m", "mcp_exec_health_gen_ai_chat.main"], env=dict(os.environ)
        ),
        anthropic.types.beta.BetaMCPServerStdioParams(
            name="mcp_visualization",
            command="python", args=["-m", "mcp_visualization_health_gen_ai_chat.main"], env=dict(os.environ)
        ),
    ]
    response = await client.beta.messages.create(
        model="claude-opus-4-8",
        max_tokens=4096,
        system=SYSTEM_PROMPT,
        messages=req.messages,
        mcp_servers=mcp_servers,
        betas=["mcp-client-2025-04-04"],
        thinking={"type": "adaptive"},
    )
    text = "\n".join(b.text for b in response.content if hasattr(b, "text"))
    return JSONResponse({"reply": text})

app.mount("/", StaticFiles(directory=str(Path(__file__).parent), html=True))
```

Start the server:

```bash
cd health-gen-ai-chat
uvicorn src.chat_agent.server:app --reload --port 8000
```

Open your browser at `http://localhost:8000` — the chat UI loads and charts are rendered inline.

## How the agent reasons

When you ask a question the agent follows a consistent pattern:

1. **Discover** — calls `list_local_metrics()` and `get_dimensions_by_semantic_model()` to understand available data.
2. **Inspect** — calls `get_table_columns(model_name)` to get exact column names.
3. **Query** — writes a SQL `SELECT` and calls `execute_read_query(sql)`.
4. **Visualize** — if the answer suits a chart, calls `generate_vega_chart(data, ...)` and wraps the result in `{"vega_spec": ...}`.
5. **Explain** — returns a plain-language explanation alongside the data or chart.

You can ask follow-up questions within the same session — the conversation history is preserved, so you can say things like "now filter that to just sensor scans" and the agent will remember the previous query context.

## Troubleshooting

### `FileNotFoundError: dbt artifact not found`

Run `dbt compile` inside `dbt_health_gen_ai_chat/`. The manifest files must exist before starting the agent.

### `RuntimeError: MYSQL_ALCHEMY_URI environment variable is not set`

Export `MYSQL_ALCHEMY_URI` before running the agent (see Step 4).

### `Execution error in MySQL database: (pymysql.err.OperationalError)`

- Check that your MySQL server is running and reachable.
- Verify the host, port, username, password, and database name in `MYSQL_ALCHEMY_URI`.
- Confirm the MySQL user has `SELECT` privileges on `cgm_abbot_connector` and `strava-to-db`.

### `anthropic.APIError: authentication_error`

Check that `ANTHROPIC_API_KEY` is set and valid.

### Charts do not render in the browser

- Ensure the browser can reach the CDN links for Vega, Vega-Lite, and vega-embed (they are loaded from `cdn.jsdelivr.net`).
- Check the browser console for JavaScript errors.
- Verify the assistant's reply contains a `vega_spec` JSON block (the UI detects it automatically).

## Project structure reference

```
health-gen-ai-chat/
├── pyproject.toml                          # Python package config & dependencies
├── dbt_health_gen_ai_chat/                 # dbt project
│   ├── dbt_project.yml
│   ├── models/
│   │   ├── source_schema.yml               # raw source table definitions
│   │   ├── semantic_schema.yml             # semantic models & metrics
│   │   ├── view_glucose_register.sql       # dbt model (view)
│   │   └── view_strava_activities.sql      # dbt model (view)
│   └── target/                             # generated by dbt compile
│       ├── manifest.json
│       └── semantic_manifest.json
└── src/
    ├── mcp_semantic_healh_gen_ai_chat/     # Semantic MCP server
    │   └── main.py
    ├── mcp_exec_health_gen_ai_chat/        # Exec MCP server
    │   └── main.py
    ├── mcp_visualization_health_gen_ai_chat/ # Visualization MCP server
    │   └── main.py
    └── chat_agent/                         # LLM agent + web UI
        ├── main.py                         # Terminal chat agent
        └── index.html                      # Web chat UI
```
