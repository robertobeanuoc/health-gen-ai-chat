# How to Use вЂ” Health Gen AI Chat

This guide walks you from a fresh checkout to a running chat session where you can ask natural-language questions about your health data and get SQL results and charts back.

## Architecture overview

```

      в–ј
```

The three MCP servers run as child processes of the chat agent. You only need to start the agent вЂ” it manages the rest.

## Step 1 вЂ” Install dependencies

```bash
cd health-gen-ai-chat

# Create the virtual environment and install all dependencies in one step
uv sync
```

`uv sync` reads `pyproject.toml`, pins Python 3.11, creates `.venv/`, and installs every dependency from the locked `uv.lock` file. No manual venv activation or `pip install` needed.

---

## Step 2 вЂ” Configure dbt

The dbt project lives in `dbt_health_gen_ai_chat/`. The repository already includes a `profiles.yml` that reads the MySQL connection from environment variables вЂ” **no credentials are hardcoded**:

```yaml
# dbt_health_gen_ai_chat/profiles.yml  (already in the repo)
dbt_health_gen_ai_chat:
  target: dev
  outputs:
    dev:
      type: mysql
      server: "{{ env_var('MYSQL_HOST') }}"
      port: "{{ env_var('MYSQL_PORT', '3306') | int }}"
      schema: "{{ env_var('MYSQL_DATABASE') }}"
      username: "{{ env_var('MYSQL_USER') }}"
      password: "{{ env_var('MYSQL_PASSWORD') }}"
      ssl_disabled: true
```

All dbt commands must include `--profiles-dir dbt_health_gen_ai_chat` so dbt finds this file instead of looking in `~/.dbt/`. Run these after setting the env vars in Step 4.

Test the connection:

```bash
uv run dbt debug --project-dir dbt_health_gen_ai_chat --profiles-dir dbt_health_gen_ai_chat
```

---

## Step 3 вЂ” Compile dbt

The semantic MCP server reads `target/manifest.json` and `target/semantic_manifest.json`. These files are generated by `dbt compile`:

```bash
uv run dbt compile --project-dir dbt_health_gen_ai_chat --profiles-dir dbt_health_gen_ai_chat
```

You should see two files created:

```
dbt_health_gen_ai_chat/target/manifest.json
dbt_health_gen_ai_chat/target/semantic_manifest.json
```

> Re-run `dbt compile` any time you change your dbt models or schema definitions.
> In Docker this step runs automatically at container startup вЂ” you do not need to run it manually.

---

## Step 4 вЂ” Set environment variables

Copy the example file and fill in your values:

```bash
cp .env.example .env
```

```bash
# .env
ANTHROPIC_API_KEY=sk-ant-...

# MySQL connection вЂ” shared by the app, mcp-exec, and dbt
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=your_user
MYSQL_PASSWORD=your_password
MYSQL_DATABASE=your_database
```

Optional вЂ” override the Claude model used by the **web UI** (terminal mode always uses `claude-opus-4-8`):

```bash
CLAUDE_MODEL=claude-haiku-4-5-20251001   # default
# CLAUDE_MODEL=claude-sonnet-4-6
# CLAUDE_MODEL=claude-opus-4-8
```

Only thinking-capable models are supported (`thinking={"type": "adaptive"}` is always enabled).

The app loads `.env` automatically via `python-dotenv` on startup.

---

## Step 5 вЂ” Run the chat agent (terminal mode)

```bash
uv run python -m src.chat_agent.main
```

You will see:

```
Health Gen AI Chat вЂ” type 'quit' to exit.

You: 
```

Type any question about your health data. The agent will:
1. Call the semantic tools to discover available models and columns.
2. Write and execute a SQL query.
3. If appropriate, generate a chart and print the Vega-Lite spec.

Type `quit`, `exit`, or `q` to stop. Press `Ctrl+C` to exit immediately.

---

## Step 6 вЂ” Use the web UI

The web UI (`src/chat_agent/index.html`) is a responsive HTML chat interface that works on desktop, tablet, and mobile вЂ” on narrow screens the session sidebar becomes a slide-in drawer. It talks to `POST /api/chat` and renders Vega-Lite charts inline. The FastAPI server (`src/chat_agent/server.py`) is included in the repo and its dependencies (`fastapi`, `uvicorn`) are already part of the project.

Start the server:

```bash
uv run uvicorn src.chat_agent.server:app --reload --port 8000
```

Open your browser at `http://localhost:8000` вЂ” the chat UI loads and charts are rendered inline.

The model defaults to `claude-haiku-4-5-20251001`; set `CLAUDE_MODEL` in `.env` to use a different thinking-capable model (e.g. `claude-sonnet-4-6` or `claude-opus-4-8`).

## Alternative вЂ” Docker

If you prefer not to install Python or uv locally, you can run the web server in a container.

### Requirements

- Docker and Docker Compose
- `.env` file in the project root (Step 4 above)
- dbt artifacts already compiled (Step 3 above)

### Build and start

```bash
docker compose up --build
```

The app is available at `http://localhost:8000`.

> **Note:** `MYSQL_ALCHEMY_URI` must use your host's IP address (or a Docker network hostname) instead of `localhost` so the container can reach the database.

## How the agent reasons

When you ask a question the agent follows a consistent pattern:

1. **Discover** вЂ” calls `list_local_metrics()` and `get_dimensions_by_semantic_model()` to understand available data.
2. **Inspect** вЂ” calls `get_table_columns(model_name)` to get exact column names.
3. **Query** вЂ” writes a SQL `SELECT` and calls `execute_read_query(sql)`.
4. **Visualize** вЂ” if the answer suits a chart, calls `generate_vega_chart(data, ...)` and wraps the result in `{"vega_spec": ...}`.
5. **Explain** вЂ” returns a plain-language explanation alongside the data or chart.

You can ask follow-up questions within the same session вЂ” the conversation history is preserved, so you can say things like "now filter that to just sensor scans" and the agent will remember the previous query context.

## Troubleshooting

### `FileNotFoundError: dbt artifact not found`

Run `dbt compile` inside `dbt_health_gen_ai_chat/`. The manifest files must exist before starting the agent.

### `RuntimeError: Missing required env vars: MYSQL_HOST вЂ¦`

Set `MYSQL_HOST`, `MYSQL_USER`, `MYSQL_PASSWORD`, and `MYSQL_DATABASE` in `.env` (see Step 4).

### `Execution error in MySQL database: (pymysql.err.OperationalError)`

- Check that your MySQL server is running and reachable.
- Verify the values of `MYSQL_HOST`, `MYSQL_PORT`, `MYSQL_USER`, `MYSQL_PASSWORD`, and `MYSQL_DATABASE`.
- Confirm the MySQL user has `SELECT` privileges on `cgm_abbot_connector` and `strava-to-db`.

### `anthropic.APIError: authentication_error`

Check that `ANTHROPIC_API_KEY` is set and valid.

### Charts do not render in the browser

- Ensure the browser can reach the CDN links for Vega, Vega-Lite, and vega-embed (they are loaded from `cdn.jsdelivr.net`).
- Check the browser console for JavaScript errors.
- Verify the assistant's reply contains a `vega_spec` JSON block (the UI detects it automatically).
