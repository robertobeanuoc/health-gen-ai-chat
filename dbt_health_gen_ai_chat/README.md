# dbt_health_gen_ai_chat

This is the dbt project that defines the data models and semantic layer for the Health Gen AI Chat system.

## Overview

The project reads from three raw MySQL sources and builds views and a semantic manifest that the MCP semantic server reads at runtime.

```
MySQL sources
  ├── cgm_abbot_connector.glucose_register   (Abbott LibreLink CGM data)
  ├── strava-to-db.strava_activities         (Strava activity data)
  └── food_recognition.food_register         (Food intake & glycemic data)
        │
        ├── models/view_glucose_register.sql      → adds timehour, timestamp_day
        ├── models/view_strava_activities.sql     → pass-through view
        └── models/view_food_register.sql         → adds timehour, timestamp_day
              │
              └── semantic_schema.yml             → semantic models & metrics
                    │
                    └── dbt compile
                          ├── target/manifest.json
                          └── target/semantic_manifest.json
```

## Project structure

```
dbt_health_gen_ai_chat/
├── dbt_project.yml          # project name, version, model config
├── profiles.yml             # connection profile — reads MySQL params from env vars
├── models/
│   ├── source_schema.yml    # raw source table & column definitions
│   ├── semantic_schema.yml  # semantic models, entities, measures, dimensions
│   ├── view_glucose_register.sql
│   ├── view_strava_activities.sql
│   └── view_food_register.sql
└── target/                  # generated by dbt compile — do not edit manually
    ├── manifest.json
    └── semantic_manifest.json
```

## Configuration

The project includes `profiles.yml` which reads all connection details from environment variables — no credentials are stored in the file:

```yaml
# dbt_health_gen_ai_chat/profiles.yml
dbt_health_gen_ai_chat:
  target: dev
  outputs:
    dev:
      type: mysql
      server: "{{ env_var('MYSQL_HOST') }}"
      port: "{{ env_var('MYSQL_PORT', '3306') | int }}"
      schema: "{{ env_var('MYSQL_DATABASE') }}"
      username: "{{ env_var('MYSQL_USER') }}"
      password: "{{ env_var('MYSQL_PASSWORD') }}"
      ssl_disabled: true
```

Set `MYSQL_HOST`, `MYSQL_PORT`, `MYSQL_USER`, `MYSQL_PASSWORD`, and `MYSQL_DATABASE` in your `.env` file (see root `.env.example`). All dbt commands require `--profiles-dir dbt_health_gen_ai_chat` so dbt finds this file instead of `~/.dbt/profiles.yml`.

In Docker the entrypoint automatically runs `dbt compile` at startup — no manual step needed.

## Common commands

All commands should be run from the repo root using `uv run` so they execute inside the project virtual environment.

```bash
# Verify the database connection and dbt setup
uv run dbt debug --project-dir dbt_health_gen_ai_chat --profiles-dir dbt_health_gen_ai_chat

# Generate manifest artifacts required by the semantic MCP server
uv run dbt compile --project-dir dbt_health_gen_ai_chat --profiles-dir dbt_health_gen_ai_chat

# Build (materialize) all models in the database
uv run dbt run --project-dir dbt_health_gen_ai_chat --profiles-dir dbt_health_gen_ai_chat

# Run schema tests
uv run dbt test --project-dir dbt_health_gen_ai_chat --profiles-dir dbt_health_gen_ai_chat

# Generate and serve documentation
uv run dbt docs generate --project-dir dbt_health_gen_ai_chat --profiles-dir dbt_health_gen_ai_chat
uv run dbt docs serve --project-dir dbt_health_gen_ai_chat --profiles-dir dbt_health_gen_ai_chat
```

> **Important:** Run `dbt compile` before starting the chat agent. The semantic MCP server reads `target/manifest.json` and `target/semantic_manifest.json` — if these files do not exist the server will fail with a `FileNotFoundError`.

## Data sources

### `abbot.glucose_register` (schema: `cgm_abbot_connector`)

Continuous glucose monitor readings from the Abbott LibreLink connector.

| Column | Type | Description |
|---|---|---|
| `uuid` | string | Unique record identifier |
| `row_created_at` | datetime | Row insertion timestamp |
| `row_updated_at` | datetime | Row last-update timestamp |
| `timestamp` | datetime | When the glucose was measured |
| `glucose_value` | double | Glucose level (mg/dL or mmol/L depending on device locale) |
| `sensor_scan` | tinyint | `1` = sensor scan, `0` = manual reading |

dbt model: `view_glucose_register` — adds two derived columns:
- `timehour` — hour of day extracted from `timestamp` (`HOUR(timestamp)`)
- `timestamp_day` — date-only value (`DATE(timestamp)`)

### `strava.strava_activities` (schema: `strava-to-db`)

Physical activity records synced from Strava.

| Column | Type | Description |
|---|---|---|
| `activity_id` | int | Unique activity identifier |
| `name` | string | Activity title |
| `activity_type` | string | General type (Run, Ride, Swim, …) |
| `sport_type` | string | Detailed sport classification |
| `start_date` | datetime | UTC start time |
| `distance_m` | double | Distance in metres |
| `moving_time_s` | int | Moving time in seconds |
| `elapsed_time_s` | int | Total elapsed time in seconds |
| `total_elevation_gain_m` | double | Elevation gain in metres |

dbt model: `view_strava_activities` — passes all columns through unchanged (`SELECT * FROM source`).

### `food.food_register` (schema: `food_recognition`)

Food intake records with glycemic and nutritional data.

| Column | Type | Description |
|---|---|---|
| `uuid` | string | Unique record identifier |
| `file_uid` | string | Identifier of the source image/file |
| `created_at` | datetime | When the food was logged |
| `food_type` | string | Name/type of the food item |
| `glycemic_index` | int | Glycemic index of the food |
| `weight_grams` | int | Weight of the portion in grams |
| `carbohydrate_percentage` | decimal | Carbohydrate content as a percentage |
| `carbohydrate_weight_grams` | decimal | Weight of carbohydrates in grams |
| `absorption_type` | string | `fast` or `slow` absorption |

dbt model: `view_food_register` — adds derived columns `timehour` (`HOUR(created_at)`) and `timestamp_day` (`DATE(created_at)`).

## Semantic layer

Defined in `models/semantic_schema.yml`. The semantic layer is consumed by the MCP semantic server to give the LLM an understanding of available measures and dimensions without it needing to know raw SQL schemas.

### `semantic_glucose_register`

| Concept | Name | Details |
|---|---|---|
| Entity | `uuid` | Primary key |
| Dimension | `timestamp_day` | Time grain: day |
| Dimension | `timestamp_hour` | Categorical (hour 0–23) |
| Dimension | `sensor_scan` | Categorical (0 or 1) |
| Measure | `glucose_value_average` | `AVG(glucose_value)` |

### `semantic_strava_activities`

| Concept | Name | Details |
|---|---|---|
| Entity | `activity_id` | Primary key |
| Dimension | `start_date` | Time grain: day |
| Measure | `distance_m_sum` | `SUM(distance_m)` |

### `semantic_food`

| Concept | Name | Details |
|---|---|---|
| Entity | `uuid` | Primary key |
| Dimension | `food_type` | Categorical |
| Dimension | `absorption_type` | Categorical (`fast` / `slow`) |
| Dimension | `glycemic_index` | Categorical |
| Measure | `weight_grams_sum` | `SUM(weight_grams)` |
| Measure | `weight_grams_average` | `AVG(weight_grams)` |
| Measure | `carbohydrate_weight_grams_sum` | `SUM(carbohydrate_weight_grams)` |
| Measure | `carbohydrate_weight_grams_average` | `AVG(carbohydrate_weight_grams)` |
